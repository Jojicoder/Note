Tailwind vs 普通のCSS

Learning Note

Tailwind CSS

CSS を
クラス名で書く
という考え方

Tailwind は CSS ファイルを書かず、HTML のクラス属性に直接スタイルを書くフレームワークです。
普通の CSS との違いを実例で比べてみましょう。

01

CSSファイル不要

HTML だけで完結。.css ファイルを別途作る必要がない。

02

クラス名を考えなくていい

.card-wrapper-inner… みたいな名前を悩む必要がない。

03

レスポンシブが簡単

sm: md: lg: をクラスに書くだけで画面サイズ対応できる。

CONCEPT

一番の違い：どこにスタイルを書くか

普通のCSS 
/* style.css */
.card {
  background: white;
  border-radius: 16px;
  padding: 24px;
  box-shadow: 0 4px 20px rgba(0,0,0,0.1);
}
.card-title {
  font-size: 18px;
  font-weight: 700;
  color: #1e293b;
}
<!-- index.html -->
<link rel="stylesheet" href="style.css">

<div class="card">
  <h2 class="card-title">タイトル</h2>
</div>
📄 2ファイル必要（HTML + CSS）
Tailwind CSS 
<!-- index.html だけでOK -->
<script src="https://cdn.tailwindcss.com"></script>

<div class="bg-white rounded-2xl p-6
            shadow-lg">
  <h2 class="text-lg font-bold
             text-slate-800">タイトル</h2>
</div>
※ CSSファイルは書かない
✅ 1ファイルだけ（HTML のみ）

RESPONSIVE

レスポンシブ対応

Tailwind の一番強い機能。クラスにプレフィックスを書くだけ。

普通のCSS — @media が必要 
.grid {
  display: grid;
  grid-template-columns: 1fr;
  gap: 12px;
}

@media (min-width: 640px) {
  .grid {
    grid-template-columns: repeat(3, 1fr);
  }
}
Tailwind — クラスだけで完結 
<div class="
  grid
  grid-cols-1
  sm:grid-cols-3
  gap-3
">
  ...
</div>

<!-- sm: = 640px 以上に適用 -->
<!-- md: = 768px 以上に適用 -->
<!-- lg: = 1024px 以上に適用 -->

HOVER / FOCUS

ホバー・フォーカス状態

普通のCSS は別セレクタが必要。Tailwind は同じ要素にそのまま書ける。

普通のCSS 
/* 別の行に書く必要がある */
.input {
  border: 2px solid #e2e8f0;
  border-radius: 8px;
  padding: 10px 14px;
}
.input:focus {
  border-color: #6366f1;
  box-shadow: 0 0 0 3px rgba(99,102,241,0.15);
  outline: none;
}
Tailwind — 同じ要素に書ける 
<input class="
  border-2
  border-slate-200
  rounded-lg
  px-4 py-2.5
  outline-none
  focus:border-indigo-500
  focus:ring-2
  focus:ring-indigo-200
" />

Tailwind

まとめ：どっちを使う？

比較項目 普通のCSS Tailwind
ファイル数 HTML + CSS（2つ） HTML だけ
クラス名 自分で考える 考えなくていい
レスポンシブ @media を別途書く sm: md: lg: だけ
ホバー・状態 .class:hover を別行 hover: を同じ行に
HTML の見た目 スッキリ クラスが長くなる
本番ファイルサイズ 書いた分だけ 使ったクラスだけ抽出（ビルド時）
おすすめ用途 小規模・学習・CSSをちゃんと学びたい プロトタイプ・React/Next.js との相性◎

💡 結論

どちらが「正解」というわけではありません。CSS の基礎を理解した上で Tailwind を使うのが理想的。 まず普通のCSSで display: flex や padding を理解すると、 Tailwind のクラス名の意味がすぐわかるようになります。
